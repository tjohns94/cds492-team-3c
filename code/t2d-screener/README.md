# T2D Risk Screener

A clinical screening tool for undiagnosed Type 2 Diabetes, trained on the Korean National Health Insurance Service (NHIS) General Health Examination dataset (2024, ~1 million patients).

End-to-end capstone project demonstrating: public-dataset ingestion, class-imbalanced modelling, probability calibration, per-prediction explainability, and a FastAPI + JavaScript serving layer вЂ” packaged so that anyone can reproduce the headline metrics from a single notebook.

---

## Quick verification

Open `notebooks/tutorial.ipynb` and run all cells. Default mode loads the committed model вЂ” flip `MODE = "train"` at the top of the config cell to retrain from scratch.

- `"pretrained"`: ~30 seconds on CPU. Loads the committed bundle from `models/` and reproduces the test metrics below.
- `"train"`: ~5-10 minutes on CPU. Retrains CatBoost with the same hyperparameters as `train.py`.

Both modes produce the same four outputs: metrics, ROC curve, reliability diagram (saved to `figures/calibration_pre_post.png`), SHAP waterfall for one patient, and a full library-call prediction demo (the same path the API wraps).

## What the app does

A clinician or user enters basic health-examination values вЂ” age, body composition, blood pressure, urine protein, smoking/alcohol status, and optionally lab values (hemoglobin, liver enzymes, kidney function, lipid panel) вЂ” and the app returns:

- A **calibrated risk probability** for undiagnosed T2D.
- A **risk tier** (Low / Moderate / High / Very High) from validation-set probability quantiles.
- A **SHAP waterfall chart** explaining which inputs drove the prediction.
- A **clinical recommendation** matched to the risk tier.

The tool identifies patients who would benefit from a fasting plasma glucose (FPG) or HbA1c test вЂ” it does not replace one.

## Running the full stack locally

**1. Install dependencies**

```bash
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

**2. (Optional) Retrain from scratch**

The committed `models/` bundle is ready to serve; `train.py` is only needed if you want to reproduce the artefacts from the raw dataset. The script downloads the ~91 MB NHIS CSV to `data/` automatically.

```bash
python train.py
```

**3. Start the API server**

```bash
uvicorn api.main:app --reload
```

Open [http://localhost:8000](http://localhost:8000).

### Reproducing figures from code

The committed PNGs in `figures/` are the authoritative versions вЂ” check them into git and use them as-is. For anyone who wants to independently verify that the figures are reproducible from the committed model and data, `scripts/` contains stand-alone generators that mirror the original training logic in `all_models_roc.ipynb`:

| Script | Reproduces | Expected runtime |
|---|---|---|
| `python scripts/generate_shap_summary.py` | `figures/shap_importance.png` | ~30 s on CPU |
| `python scripts/generate_confusion_matrix.py` | `figures/catboost_confusion_matrix.png` | ~30 s on CPU |
| `python scripts/generate_roc_comparison.py` | `figures/roc_all_models.png` + `scripts/roc_comparison_data.json` | ~5вЂ“10 min on CPU; `torch` required for the PyTorch Tabular Net row (the script skips it gracefully otherwise) |

Each script downloads the NHIS dataset to `data/` on first run. Running a script overwrites the corresponding PNG. `figures/screener_ui.png` is a manual screenshot and has no generator.

## Limitations (short version)

- Korean population, 2024 cohort only. Generalisation untested.
- Label is FPG-based; does not separate newly undiagnosed from poorly controlled known T2D.
- Missing lab values act as a weak "healthy" signal because of selection bias in who gets labs ordered. Submissions with zero lab values will underestimate risk; the frontend surfaces a warning in that case.
- Research capstone project. Not clinically validated.

Full discussion: [`MODEL_CARD.md`](MODEL_CARD.md).
